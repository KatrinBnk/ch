
**Перед запуском (или открытием githubPages) запустите сервер с бекендом.**

### Возможны два варианта просмотра
1. Перейти по последней (актуальной) ссылке в deployments. При переходе может выйти предупреждение о том, что сайт не безопасен -- просто игнорируем его.
2. Запустить локально на своем устройстве
    1. Склонировать репозиторий
    2. Выполнить установку пакетов ``` npm install ```
    3. Выполнить запуск ``` npm run ```

Для того, чтобы деплоить на pages необходимо было указать префикс в адресе страницы по названию репозитория (в данном случае "ch"), если будет необходимость или желание изменить, то это можно сделать быстро в src/main.js
```js
const router = createRouter({  
    history: createWebHistory('/ch'), //Базовый адрес (префикс)
    routes  
});
```

### Просьба к серверной части:
1. Открыть доступ к маршруту с просмотром услуг, так как возможность просмотра списка услуг необходима и для неавторизованных пользователей (возможны и другие варианты решения этой проблемы)  
```java
.requestMatchers(HttpMethod.GET, "/usluga/**").permitAll()
```
2. В маршрутах "get user/{email}" и "get applications/{userId}" **вылетает 500 ошибка** от сервера из-за того, что невозможно однозначно определить маршруты. **Сейчас я временно изменяла эти маршруты для точного определения на "get user/email/{email}" и "get applications/master/{userId}" соответственно.** *В случае внесения нормальных изменений и невозможностью связаться со мной обновить этот момент можно самостоятельно в src/service/scheduleService.js*
```js
const response = await axios.get(`http://localhost:8080/applications/master/${userId}`, {  
    headers: {  
        Authorization: token,  
    },  
});
```
